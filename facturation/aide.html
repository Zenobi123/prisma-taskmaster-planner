👋 Bienvenue dans PRISMA GESTION

PRISMA GESTION est une application web de gestion commerciale et fiscale destinée aux cabinets de comptabilité et de fiscalité du Cameroun. Elle couvre tout le cycle de vie d'un dossier client : prospect → contrat → devis → facture → reçu → suivi fiscal annuel.

Caractéristiques clés :
  • Application 100% locale dans votre navigateur — aucune donnée n'est envoyée sur Internet.
  • Toutes vos données sont stockées dans le navigateur (localStorage) — pensez à exporter régulièrement.
  • Fonctionnement hors-ligne complet (sauf l'export PDF qui nécessite Internet).
  • Adapté aux termes fiscaux camerounais : IGS, Patente, DSF, DARP, DBEF, CFLP, CIME, DGE, etc.

🚀 Démarrage rapide

Les 5 premières étapes recommandées pour démarrer sereinement :

  1. 1

    Configurer le cabinet

    Allez dans Paramètres Cabinet et renseignez le nom du signataire, son titre, et téléversez votre signature + cachet (formats PNG/JPG conseillés).

  2. 2

    Créer vos premiers clients

    Module Clients → bouton « Ajouter un client ». Si vous avez déjà un fichier Excel/CSV, utilisez l'import (modèle disponible).

  3. 3

    Activer la gestion externalisée

    Pour chaque client que votre cabinet prend en gestion, mettez « Gestion externalisée = Oui » dans sa fiche. Il apparaîtra alors dans le module Gestion.

  4. 4

    Émettre votre premier document

    Devis pour démarrer une relation, facture pour facturer une prestation, reçu pour valider un encaissement, courrier pour échanger avec le client ou l'administration.

  5. 5

    Exporter vos données régulièrement

    Une icône de sauvegarde apparaît sous la barre de navigation. Pensez à exporter au moins une fois par semaine pour ne rien perdre.

🧩 Les modules en détail

PRISMA GESTION s'articule autour de 12 modules accessibles depuis la page d'accueil. Voici leur rôle.

👥 Clients

CRUD complet des clients (personnes morales et physiques) avec NIU, centre fiscal CFLP, régime d'imposition et situation immobilière.

  • Import CSV / Excel / JSON avec modèle téléchargeable.
  • Export JSON (bouton « Exporter » de l'en-tête) — deux portées :
    • Liste complète : tous les clients, avec toutes leurs données (agences, régime, immobilier…) — format fidèle et réimportable.
    • Client individuel (icône d'export sur chaque ligne) : le client + tout son historique d'opérations (factures, devis, reçus, propositions, notes, courriers, contrats).
  • Réimporter un export individuel restaure aussi l'historique (fusion sans doublon). Une sauvegarde de sécurité est créée avant l'import.
  • Filtrage du centre des impôts selon la ville et le régime : un contribuable IGS à Yaoundé verra seulement les CFLP de Yaoundé. Un contribuable Réel verra les CIME / CSI / DGE.
  • Calcul automatique des impôts : IGS (par classe + réduction CGA), Patente, TDL, Licence boissons, Solde IR.
  • Calcul automatique des impôts immobiliers : PSL, Bail, TPF (par bien si multi-agences).
  • Multi-agences : un client peut avoir plusieurs établissements ; les CA et impôts sont cumulés/ventilés selon les besoins.
  • Fiche PDF : le bouton « Fiche PDF » de chaque ligne ou « Télécharger PDF » dans le formulaire affiche un modal professionnel avec miniature de la première page, résumé et contrôle qualité, puis génère et télécharge directement une fiche A4 récapitulative, comme le téléchargement des factures.
  • Corbeille : un client supprimé est placé dans la corbeille (bouton en haut de page) et peut être restauré avec tout son historique (factures, devis, reçus…) ou supprimé définitivement.
💡 Astuce : supprimer un client est réversible. Tant qu'il est dans la corbeille, ses documents (factures, devis, reçus, contrats, courriers) ne sont jamais effacés et lui sont rendus dès la restauration.

📄 Devis / Proforma

Établissement de devis convertibles en factures. Numérotation automatique DEV-XXXX/AAAA ou saisie manuelle (mode Auto/Manuel).

Pour les clients multi-agences, un bouton d'ajout est généré par bien pour les impôts immobiliers — le libellé inclut la ville et le quartier (ex. Bail_Yaoundé/Bastos).

🧾 Factures

Émission des factures avec classification systématique de chaque ligne en « Impôt » ou « Honoraire ». Cette distinction est fondamentale pour le suivi ultérieur (paiement des impôts ≠ rémunération du cabinet).

Mode Auto/Manuel pour le numéro et la date — pratique pour reprendre des factures déjà émises sur papier.

💼 Propositions de paiement

Établissement de propositions consolidées (Impôts + Honoraires) à présenter au client avant facturation. Sert souvent de support au pré-engagement.

🧮 Reçus de paiement

Reçus avec conversion automatique du montant en lettres (orthographe française classique). Numérotation RECU-XXXX/AAAA.

Les paiements sont appliqués aux factures du client chronologiquement, avec distinction Impôts / Honoraires — le statut de chaque facture (émise / partielle / payée) est calculé dynamiquement.

✉ Gestion du Courrier

Édition de courriers avec modèles prédéfinis et suivi d'envoi. Trois catégories :

  • Clients : lettre de mission, relance, mise en demeure, transmission de documents, lettre d'information.
  • Administratifs : demande NIU, changement de régime, déclaration de cessation d'activité.

✅ Missions Nouveau v2.4

Crée et suit les missions confiées aux collaborateurs pour le compte des clients (ex. : « Audit fiscal 2026 », « Déclaration DSF », « Réunion de cadrage »).

  • Statuts automatiques selon les dates : En attente → En cours → En retard. La transition vers Terminée reste manuelle.
  • Ordre de mission : un clic génère 3 exemplaires (missionnaire / superviseur / contribuable) qui apparaissent dans le module Courrier.
  • Rapport de mission : upload JSON / MD / TXT avec aperçu. À la soumission, 2 courriers sont automatiquement produits — version superviseur (avec anomalies) + version client (expurgée).
  • Pagination 10/page, tri par priorité de statut, masquage auto des missions terminées depuis plus de 30 jours.

📅 Planning Nouveau v2.4

  • Filtre par collaborateur en haut à droite.

📚 Concepts métier

Les notions fiscales et comptables que l'application manipule, expliquées simplement.

SigleNom completCible

Modèle multi-agences

Un client peut exercer sur plusieurs établissements (agences). Chacun a son propre libellé, sa localisation (ville/quartier), son CA, son statut immobilier (locataire / propriétaire), son loyer et la valeur du bien.

  • IGS / Patente / TDL / Licence : calculés sur le CA cumulé (un seul contribuable, un seul régime).
  • PSL / Bail / TPF : calculés par bien puis cumulés.
  • L'agence principale (siège) est marquée ★ et hérite par défaut de la localisation du client.

Classification Impôt / Honoraire

Toutes les lignes de facture sont systématiquement classifiées en Impôt (obligation fiscale réglée au Trésor public) ou Honoraire (prestation intellectuelle du cabinet). Cette distinction permet :

  • D'isoler les sommes effectivement perçues par le cabinet (honoraires) du flux client → Trésor (impôts).
  • De suivre dans le module Gestion le règlement de chaque impôt (cabinet ou client direct).
  • De produire des états distincts pour la gestion financière du cabinet.

💾 Sauvegardes et protection des données

Vos données sont stockées dans le navigateur (localStorage). En cas d'effacement du cache, de panne disque ou de réinitialisation, elles peuvent être perdues. L'application intègre plusieurs mécanismes pour les protéger.

Snapshot avant opérations destructives

Avant chaque opération risquée (import en masse, suppression, migration), l'application prend automatiquement un instantané (_prisma_backup_snapshot) que vous pouvez restaurer.

Export / Import JSON

Un bandeau de sauvegarde apparaît sous la barre de navigation : il permet d'exporter toutes vos données dans un fichier prisma-gestion-backup-AAAA-MM-DD_HHMM.json, et de réimporter ce fichier sur un autre navigateur / une autre machine.

Lecture défensive (safeParseJSON)

Si une donnée stockée devient corrompue (JSON invalide), l'application la sauvegarde sous une clé _prisma_backup_corrupted_* avant d'utiliser une valeur par défaut — vos données ne sont jamais effacées silencieusement.

⚠ Bonnes pratiques : exportez au moins une fois par semaine, conservez plusieurs versions horodatées, et stockez les sauvegardes en dehors du navigateur (clé USB, cloud).

🖨 Impression et export PDF

Tous les documents (factures, reçus, devis, propositions, notes, courriers) s'impriment directement via le menu d'impression du navigateur — aucune fenêtre supplémentaire ne s'ouvre.

Modèle d'impression unifié

Format A4 avec marges 1,5 cm, en-tête cabinet, pied de page automatique, signature et cachet présents si configurés dans les Paramètres Cabinet.

Export PDF

Bouton « Télécharger PDF » dans chaque module : utilise jsPDF et html2canvas (chargés via CDN cdnjs). Internet requis pour cette fonctionnalité spécifique.

💡 Pour un PDF de meilleure qualité, utilisez plutôt la fonction « Imprimer » du navigateur (Ctrl+P) puis « Enregistrer en PDF ».

💡 Astuces et raccourcis

  • Recherche dans la liste clients : tapez dans le champ recherche — il filtre par nom, NIU, CFLP, ville, contact.
  • Édition rapide d'un client : dans la liste clients, cliquez sur l'icône crayon ; l'écran d'édition réutilise le formulaire de création avec les valeurs chargées.
  • Mode Auto/Manuel : pour reprendre un document déjà émis sur papier, basculez sur le mode Manuel et saisissez votre propre numéro et date.
  • Synchronisation entre onglets : si vous ouvrez plusieurs onglets de l'app, les modifications sont synchronisées en temps réel via l'événement storage.
  • Impression A4 : utilisez Ctrl+P dans n'importe quel module pour imprimer la page courante avec le style optimisé.
  • URL de partage pour le module Gestion : copiez l'URL complète après avoir sélectionné un client — elle inclut ?client=...&tab=fiscal&exercice=2026 et reconduit votre interlocuteur au même endroit.
  • Auto-save : les formulaires longs (clients, gestion) sauvegardent automatiquement votre saisie en cas de rafraîchissement involontaire.

❓ Questions fréquentes

Mes données disparaissent quand je vide le cache. Pourquoi ?

Toutes les données sont stockées dans localStorage, qui fait partie du cache du navigateur. Vider le cache → données effacées. Solution : exportez régulièrement via le bouton « Exporter les données » du bandeau de sauvegarde.

Comment partager mes données entre deux ordinateurs ?

Exportez un fichier JSON depuis l'ordinateur source, transférez-le (clé USB, cloud), puis importez-le sur l'ordinateur cible via le même bandeau. Veillez à n'avoir qu'un seul utilisateur actif à la fois pour éviter les conflits.

J'ai supprimé un client par erreur. Comment le récupérer ?

Ouvrez le module Clients → bouton « Corbeille » en haut de page (un compteur indique le nombre d'éléments). Repérez le client et cliquez sur « Restaurer » : il réintègre la liste active avec tout son historique (factures, devis, reçus, contrats, courriers). La suppression d'un client est donc réversible tant que vous ne l'avez pas supprimé définitivement depuis la corbeille.

Où sont stockées les pièces jointes du module Gestion ?

Deux cas. Si vous avez connecté le dossier de documents (bannière en haut du module Gestion, Chrome/Edge uniquement) : les fichiers sont écrits sur le disque dans documents/<client>/<exercice>/… à l'intérieur du dossier choisi — ils sont visibles dans l'explorateur Windows et doivent être sauvegardés avec le dossier de l'application (l'export JSON ne contient que leurs références, pas leur contenu). Sinon : les pièces sont encodées dans le stockage interne du navigateur (limité à 5 Mo par fichier) et incluses dans l'export JSON. L'icône devant chaque pièce vous renseigne : 💾 = fichier sur disque, 📎 = stockage navigateur.

Le navigateur me redemande l'accès au dossier de documents à chaque ouverture.

C'est une protection du navigateur : l'accès en écriture à un dossier du disque doit être reconfirmé par session. Cliquez sur « Réactiver l'accès » dans la bannière orange. Sur les versions récentes de Chrome/Edge, la fenêtre de permission propose une option « Autoriser à chaque visite » qui supprime cette demande.

Mon client n'apparaît pas dans le module Gestion. Pourquoi ?

Le module Gestion ne liste que les clients dont la fiche indique « Gestion externalisée = Oui ». Allez dans Clients → éditer la fiche → cocher la case → enregistrer.

J'ai des anciennes valeurs « CDI YDE 1 » dans mes fiches. Que faire ?

L'application migre automatiquement ces valeurs vers la nomenclature DGI 2026 (CFLP Yaoundé 1) au prochain chargement du module Clients. Une sauvegarde snapshot est prise avant migration — aucune donnée perdue.

Le centre fiscal de mon client est introuvable dans la liste.

Choisissez l'option « Autre (saisie libre) » en bas du dropdown et saisissez le nom exact. Cela peut arriver pour des centres très récents ou pour la région Nord (Garoua) qui n'est pas listée dans la cartographie DGI 2026 consultée.

L'export PDF ne fonctionne pas.

L'export PDF utilise les bibliothèques jsPDF et html2canvas chargées via Internet (CDN). Vérifiez votre connexion. Alternative : utilisez l'impression du navigateur (Ctrl+P) puis « Enregistrer en PDF ».

Comment réinitialiser entièrement l'application ?

⚠ Action destructive. Exportez d'abord vos données. Ensuite, dans les outils développeur du navigateur (F12 → Application → Local Storage), supprimez les entrées commençant par votre domaine. Au prochain chargement, l'app sera vide.

Une donnée est apparemment corrompue. Comment la récupérer ?

L'application conserve les données corrompues sous des clés _prisma_backup_corrupted_* dans le localStorage. Accédez-y via F12 → Application → Local Storage. Vous pouvez aussi restaurer le dernier snapshot via la fonction Restaurer la sauvegarde.
